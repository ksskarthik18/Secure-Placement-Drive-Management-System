Security Theory & Implementation

1. Security Levels Implemented

  • Network Level: Not applicable (Localhost), but design supports HTTPS.
  • Application Level: RBAC, Input Validation, Session Management.
  • Data Level: AES-256 Encryption for stored files, Salted Hashing for passwords.

2. Attacks & Countermeasures

Attack Vector Countermeasure Used Status
Brute Force Account locking (Concept) / Strong Password Hashing (bcrypt is slow) Secured
Replay Attack OTP with strict expiration time (Time-Based) Secured
Man-in-the-Middle (MITM) Encrypted communication (HTTPS - Required in Prod) + E2E Encryption for Resumes Partial (Local)
SQL Injection Use of ORM (SQLAlchemy) prevents raw SQL queries. Secured
Unauthorized Access Role-Based Access Control (RBAC decorators) Secured
Data Tampering Digital Signatures (SHA-256 + RSA) for Placement Results Secured
